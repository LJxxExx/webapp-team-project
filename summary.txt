[웹 애플리케이션 프로젝트 구조, 기능 및 데이터 흐름 요약]

====================================================================================================
1. 백엔드 (Backend - Python / FastAPI / SQLAlchemy)
====================================================================================================
백엔드는 데이터베이스와 통신하며 클라이언트(React)에 REST API를 제공합니다.

■ 1.1 backend/main.py (메인 서버 및 API 라우터)
- 주요 메서드 및 기능:
  - lifespan(app): 서버가 켜질 때 DB 엔진을 연결하고 테이블을 생성(create_all)합니다. 컬럼이
    비어있다면 추가(ensure_lecture_schema)하며, 초기 더미 데이터를 주입(bulk_insert, seed)하는
    생명주기 관리 메서드입니다.
  - resolve_lecture(): 클라이언트가 보낸 학수번호(코드-분반) 또는 DB ID를 이용해 Lecture
    테이블에서 강의 객체를 검색합니다.
  - find_time_conflict(): 사용자의 기존 수강 시간표와 새로 신청하려는 강의의 요일/시간을 비교해
    겹치는지 검증합니다.
  - get_all_lectures(): 전체 강의 카탈로그를 조회합니다. (App.js 초기 구동 시 사용)
  - get_user_timetable(), enroll_lecture(), drop_lecture(): 사용자의 수강 내역(Enrollment)을
    조회/생성/삭제합니다.
  - get_user_assignments(), create_user_assignment(), update_assignment(),
    delete_assignment(): 사용자 과제(Assignment) 데이터를 CRUD 처리합니다.
- 데이터 흐름: 클라이언트(Axios) 요청 → 라우터 → SQLAlchemy DB 세션(get_db) → DB 트랜잭션 수행 →
  결과를 JSON으로 응답.

■ 1.2 backend/bulk_insert.py (데이터 동기화 스크립트)
- 주요 메서드:
  - insert_lectures(): lectures.json을 읽어와 DB에 갱신하는 진입점입니다.
  - apply_lecture_fields(), replace_meetings(): JSON 데이터를 Lecture 및 LectureMeeting 모델
    객체 필드에 매핑하고 관계 테이블 데이터를 삽입합니다.
  - remove_stale_lectures(): JSON 파일에 없는 오래된 강의 데이터를 DB에서 삭제합니다.

■ 1.3 backend/seed.py (초기 테스트 데이터 세팅)
- 주요 메서드:
  - seed_db(): DB가 비어있을 때 작동하며, pwdlib로 비밀번호를 해싱해 테스트 계정(20220001)을
    생성하고 초기 과제 2개를 등록합니다.


====================================================================================================
2. 프론트엔드 (Frontend - React / JS, JSX)
====================================================================================================
프론트엔드는 전역 상태(App.js)를 기반으로 각 하위 컴포넌트로 데이터를 내려보내는 단방향
(Top-Down) 데이터 흐름을 따릅니다.

■ 2.1 frontend/src/App.js (최상위 레이아웃 및 상태 관리)
- 주요 상태(State): page(현재 화면), user(유저 정보), lectureCatalog(전체 강의),
  assignments(과제목록), savedPlans(1안/2안 시간표).
- 주요 메서드:
  - useEffect(): 앱 실행 시 백엔드에서 강의 목록을 가져오고, 로그인 시 fetchUserData()를 호출해
    시간표와 과제 목록을 전역 상태로 저장합니다. 또한 localStorage에서 로그인 상태를 복원하여
    브라우저 새로고침 시에도 로그인을 유지합니다.
  - handleLoginSuccess(): 로그인 성공 시 사용자 정보를 localStorage에 저장합니다.
  - logout(): 로그아웃 시 상태를 초기화하고 localStorage에서 사용자 정보를 삭제합니다.
  - addAssignment(), updateAssignment(), deleteAssignment(), toggleAssignmentComplete(): 백엔드
    API를 호출해 DB를 수정한 뒤, 성공하면 전역 상태 assignments를 업데이트하여 리렌더링을 유발합니다.
  - navigateTo(): 애니메이션 효과와 함께 페이지 컴포넌트를 전환합니다.
- 데이터 흐름: 단일 진실 공급원(SSOT) 역할을 수행. 데이터 조회 및 변경은 App.js가 전담하며, 하위
  컴포넌트(Timetable, AssignmentPage 등)에는 상태(Props)와 상태 변경 콜백 함수만 넘겨줍니다.

■ 2.2 frontend/src/components/timetable/Timetable.jsx (시간표 관리)
- 주요 메서드:
  - addLecture(), deleteLecture(): 선택한 강의를 시간표 courses 배열에 추가/삭제합니다.
  - hasConflict(): 강의 추가 시 기존 시간표 목록과 겹치는 요일 및 시간이 있는지 검증합니다.
  - updateActivePlan(): 시간표가 수정되면 App.js에서 받은 setSavedPlans 콜백을 호출해 전역 상태에
    반영시킵니다.
  - assignColorsToPlanEntries(): 같은 과목에 일관된 고유 색상을 자동 배정합니다.
- 데이터 흐름: App.js에서 전체 강의(lectureCatalog)와 수강 계획(savedPlans)을 props로 받아
  내부적으로 전공/교양 필터링 및 검색 기능을 거쳐 화면에 출력합니다. 변경 사항은 즉시 전역 상태로
  올라가 학점 계산기 등에 동기화됩니다.

■ 2.3 frontend/src/components/assignment/AssignmentPage.js (과제 캘린더 및 상세 폼)
- 주요 메서드:
  - selectDate(), goBackToCalendar(): 캘린더 모드와 특정 날짜 상세 모드를 전환합니다.
  - handleSubmit(): 입력된 폼 데이터를 모아 App.js의 onAddAssignment 혹은 onUpdateAssignment를
    호출합니다.
  - getRiskInfo(), getDateStatus(): 마감일(D-day)과 체크리스트 수행률 등을 계산해 진행률과 위험도
    (안전/주의/긴급)를 시각화합니다.
- 데이터 흐름: 전달받은 assignments 상태를 분석해 달력 날짜별로 렌더링하고, 데이터를 추가/수정/삭제하는
  이벤트 발생 시 부모(App.js)의 콜백을 통해 처리합니다.

■ 2.4 frontend/src/components/grade/GradeCalculator.js (학점 및 위험도 분석기)
- 주요 메서드:
  - calcRisk(): 과목별 출석률(결석 횟수 기반), 과제 제출률, 예상 시험 점수를 종합하여 위험 점수(0~100)를
    산출하고 예상 학점(A+ ~ F)을 계산합니다.
  - updateStat(): 사용자가 입력한 결석 횟수나 시험 점수를 컴포넌트 로컬 상태(stats)로 관리합니다.
- 데이터 흐름: 전역 상태의 시간표 강의 목록(savedLectures)과 과제 목록(assignments)을 넘겨받아
  연동합니다. '과제 제출률'은 과목명과 과제명이 일치하는 데이터를 찾아 자동 계산하며, 시험 점수는 사용자
  입력을 로컬 상태로 받아 즉시 분석 결과에 반영합니다.

■ 2.5 frontend/src/components/enrollment/EnrollmentPage.js (수강신청 시뮬레이션)
- 주요 메서드:
  - generateCode(): 매 행동마다 보안 문자 2자리를 난수로 갱신합니다.
  - handleAction(): 보안 문자를 검증하고, 수강신청 목록(practiceEnrolledIds)에 과목을 추가/삭제합니다.
- 데이터 흐름: App.js의 시간표 데이터를 기반으로 연습용 신청 가능 과목 목록을 구성합니다. 백엔드 실제 DB에는
  저장되지 않으며 컴포넌트 로컬 상태에서 시뮬레이션만 수행됩니다.

■ 2.6 frontend/src/components/bar/Sidebar.js & Navbar.js
- 주요 기능:
  - Sidebar: 과제 리스트를 받아와 아직 제출하지 않은 과제 중 남은 기한 순으로 정렬하여 렌더링합니다. 특정
    과제 클릭 시 App.js를 거쳐 AssignmentPage의 해당 날짜 화면으로 바로 이동시킵니다.
  - Navbar: 선택된 탭 정보에 따라 App.js의 onNavigate를 호출해 화면을 전환합니다.

■ 2.7 frontend/src/components/academic/AcademicSection.jsx
- 주요 기능:
  - useEffect()를 활용해 3초 주기의 타이머(setInterval)를 설정하고, 현재 시간표에 있는 과목들
    (enrolledCourses)을 순환하며 학술 논문 더미 데이터를 캐러셀 형태로 노출합니다.

■ 2.8 frontend/src/components/LoginModal.js & SignupModal.js (로그인/회원가입 모달)
- 주요 기능:
  - LoginModal: 학번과 비밀번호를 입력받아 백엔드 로그인 API를 호출합니다. 로그인 성공 시 App.js의
    handleLoginSuccess 콜백을 통해 사용자 정보를 전역 상태 및 localStorage에 저장합니다.
  - SignupModal: 이메일, 학번, 학과, 비밀번호를 입력받아 회원가입 API를 호출합니다.
  - 두 모달 모두 입력 중 외부 영역 클릭 시 확인 메시지를 띄워 실수로 입력 데이터가 사라지는 것을 방지합니다.

■ 2.9 frontend/src/components/mypage/MyPage.js (마이페이지)
- 주요 기능:
  - 로그인한 사용자의 정보(이름, 이메일, 학번, 학과, 학년, 학기)를 표시합니다.
  - 학년 변경 시 onUpdateUser 콜백을 통해 백엔드에 프로필 업데이트 요청을 보냅니다.
  - 비로그인 상태에서는 로그인 버튼을 표시하여 LoginModal을 열 수 있습니다.
